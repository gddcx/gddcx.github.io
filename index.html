在Winform C++中使用

发表于 2018-11-30 | 分类于 MySQL

环境

在VS中使用C++开发Winform
使用阿里云服务器上搭建的数据库，数据库是64位的MySQL8
当然在本地也要下载MySQL，因为需要用到其中的头文件和库文件
因此VS中项目平台也要选择64位：

右击选择 属性

没有x64平台的话就点击 配置管理器 进行新建

在 项目->属性 中进行必要的配置


选择 C/C++->常规，在 附加包含目录中添加 MySQL/include 的路径

选择 链接器->常规 在 附加库目录中添加 MySQL/lib 路径

选择 链接器->输入 在附加依赖项中 输入 libmysql.lib

最后从MySQL/lib中取出libmysql.dll放在对应的x64/debug目录下（会有几个x64文件夹，放在.exe文件所在的那个）
基本到这里就完成了，不过我运行的时候还是很多莫名的错误：

错误

找不到libmysql.dll：
我认为是版本问题，有可能vs识别不了MySQL8，因为我又下载了一个MySQL5，用MySQL5的libmysql.dll覆盖掉之前设置的dll就能运行了。
找不到一大堆.lib文件：
是因为平台换到了x64，很多x64的库都找不到。
解决办法是，在c:/Program Files中找到缺少的库文件（要注意是不是x64的），然后在 链接器->常规->附加库目录 中添加库文件路径就可以了。

基本用法

这是我的一个课程作业，所以只使用了一些基本的用法。
首先:

#include "mysql.h"

然后声明一个MYSQL变量:

MYSQL mysql = new MYSQL;

然后初始化mysql：

mysql_init(&mysql);

我在网上看到一篇文章认为，上面声明变量以及初始化时mysql的使用是正确的，相反下面的写法被认为是错误的：

MYSQL *mysql = new MYSQL;
mysql_init(mysql);

不过我开发这个winform使用的是托管类,第一种定义方法会报错，我使用了第二种，而且完全能够运行。
然后连接:

if(mysql_real_connect(mysql,host,username,passwd,database,port,NULL,0))

port除非你设置了别的值，否则一般是3306，其余各个变量的意思都很清楚了。连接成功则返回true。
设置编码，当然如果你传输的全是ASCII编码字符就可以不设置了，我创建表时没有设置编码，中文默认是GBK：

mysql_query(mysql,"SET NAMES ’GBK‘")；

然后执行SQL语句:

int res = mysql_query(mysql,SQL语句);

如果执行成功返回0，执行失败返回非0
查询结果：

MYSQL_RES * result = mysql_store_result(mysql);

返回的结果是一个MSQL_RES*类型的值，可以直接用if(result)判断是否为空
获取结果：

while(MYSQL_ROW sql_row = mysql_fetch_row(result))

每次执行while循环会返回数据表中的一条记录，可以把sql_row当做数组来使用(sql_row[0],sql_row[1]….)
就完了！

当然还有一些小问题，例如声明变量的时候，如果用上面的方法

MYSQL *mysql = new MYSQL;

我会出现的错误是：…..内存已损坏……大概就是内存溢出，当我改成:

MYSQL *mysql = new MYSQL[2];

就不会出现这个问题。
还有一个问题就是，当我出现大量调用 创建连接 时，程序也会出现内存溢出的问题，网上一些回答是使用：

mysql_free_result(result);

来解决，但是对我而言并没有效果，相反还会导致程序莫名退出
我的程序需要频繁地建立连接，运行一段时间之后还会报 内存损坏 ，不过现在无端端又好了，软件也靠不住啊~
不过还是在网上找了点关于这个问题的说法，大概就是接受的数据太多，处理不来，导致内存堆积，就溢出了….我也不确定
不过还是记下一个方法，方便以后查看：

mysql_close(mysql);
mysql_library_end();

据说前一步并没有完全释放内存，需要再添加后一条语句才能完全释放。

Windows和Linux 互传文件

发表于 2018-10-23 | 分类于 琐碎

Windows和Linux服务器互传文件

今天准备在阿里云上部署后端，因为是新创建的实例，还没有jdk，就wget了一个，结果很坑啊！
wget下来的包根本就解压不了！

网上说，是因为Oracle下载jdk时需要先选择Accept，所以用wget就出问题了（不清楚为什么啊）。
所以只好现在linux下载再上传到linux服务器咯
linux和linux之间传文件很简单，使用scp命令就可以了，但是windows和linux之间传文件就触及我的知识盲区了！
赶紧学习了一下

使用XShell6：

1.先连接到你的服务器
2.点击新建文件传输，也可以之间用ctrl+alt+F，会弹出界面要你下载xftp点击取消就ok了

这时sftp:后面会有一个目录，就是你账号现在所在的目录，文件会传到这个目录来
3.输入lcd，就会弹出一个文件选择界面，选择你将要上传的文件的目录

4.输入

put jdk-8u191-linux-x64.rpm

put 后接你要上传的文件名（目录刚才已经选择了），就会将文件put到sftp后面的目录啦

5.等待！ok

使用winSCP

这个操作就很简单了，下载一个winSCP，安装一下
选择sFTP文件协议，输入主机、端口、用户、密码

连接成功后，只要把文件拖来拖去就可以实现上传或者下载了

MySQL重置密码

发表于 2018-10-12 | 分类于 MySQl

Windows 上安装的MySQL重置密码

重置密码的方法比较粗暴，其实就是重新生成data文件夹,所以需要先对数据进行备份，步骤如下：

删除my.ini文件指定的data文件夹

在命令行窗口进入MySQL目录下的bin文件夹，输入命令 

mysqld --initialize --console

命令完成后会重新生成data文件夹，命令行窗口会给出初始化密码

这时MySQL是没有启动的，启动MySQL 

net start mysql

可以用命令查看MySQL的状态

sc query mysql

然后是登录MySQL，

mysql -uroot -p

输入刚才得到的初始密码就ok了

这时候你还不能使用MySQL，需要先更改密码才行

有两种命令:

第一种是

SET PASSWORD FOR user@host = PASSWORD('new password')

第二种是

ALTER USER user IDENTIFIED BY 'new password'

我安装的是MySQL-8，使用第一种命令会报错，只能使用第二种命令，而我服务器上的MySQL-5就只能使用第一种命令

完成所有操作后，密码就重置成功啦~
